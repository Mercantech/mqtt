01 Hvad er MQTT?

MQTT er en letvægts publish/subscribe-protokol designet til enheder med begrænset båndbredde og strøm — typisk IoT-sensorer, aktuatorer og gateways. Protokollen kører over TCP/IP og bruger en broker som mellemmand mellem afsender og modtager.

📡

Publish

En klient sender en besked til et topic via brokeren.

📬

Subscribe

En klient lytter på et eller flere topics og modtager beskeder.

🔀

Broker

Central server der router beskeder mellem publishers og subscribers.

Hvorfor MQTT til IoT?

  • Minimal overhead — beskeder kan være få bytes
  • Decoupling — publishers kender ikke subscribers (og omvendt)
  • Last Will & Testament — broker sender besked hvis klient dør uventet
  • Retain — broker gemmer seneste besked på et topic til nye subscribers
  • Skalerbar — millioner af enheder kan køre mod én broker

02 Arkitektur

I modsætning til HTTP's request/response-model er MQTT event-drevet. Enheder publicerer data når det sker — de poller ikke en server.

CONNECT → SUBSCRIBE → PUBLISH flow

  1. CONNECT — Klient opretter forbindelse til broker (med client ID)
  2. SUBSCRIBE — Klient tilmelder sig topics den vil lytte på
  3. PUBLISH — Klient sender payload til et topic
  4. PINGREQ/PINGRESP — Keep-alive for at holde forbindelsen åben
  5. DISCONNECT — Graceful afslutning

04 Quality of Service (QoS)

QoS definerer garantien for levering af beskeder mellem klient og broker.

QoS 0 At most once — Fire and forget. Ingen bekræftelse. Hurtigst, men besked kan gå tabt.
QoS 1 At least once — Bekræftelse (PUBACK). Besked leveres mindst én gang — kan komme som duplikat.
QoS 2 Exactly once — 4-trins handshake (PUBREC/PUBREL/PUBCOMP). Garanteret præcis én levering. Langsomst.

Hvornår bruger man hvad?

  • QoS 0 Temperaturmålinger hvert 5. sekund — tab er OK
  • QoS 1 Alarmer, dør-events — vigtigt men duplikat OK
  • QoS 2 Betalingsdata, kritiske kommandoer — ingen duplikater

06 Arduino Opla

Arduino Opla har indbygget WiFi og kan bruge PubSubClient-biblioteket til MQTT. Her er et grundlæggende eksempel der publicerer temperatur. 

#include <WiFiS3.h>
#include <PubSubClient.h>

const char* ssid     = "DIT_WIFI";
const char* password = "DIT_PASSWORD";
const char* mqtt_server = "192.168.1.100";  // Din PC's IP
const int   mqtt_port   = 1883;

WiFiClient wifiClient;
PubSubClient client(wifiClient);

void setup() {
  Serial.begin(9600);
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) delay(500);

  client.setServer(mqtt_server, mqtt_port);
  client.connect("opla-sensor-01");
}

void loop() {
  if (!client.connected()) client.connect("opla-sensor-01");

  float temp = 22.5;  // Læs fra rigtig sensor
  char msg[16];
  dtostrf(temp, 4, 1, msg);
  client.publish("skole/rum/101/temp", msg);

  client.loop();
  delay(5000);
}

Installér biblioteker i Arduino IDE

  • PubSubClient af Nick O'Leary (via Library Manager)
  • WiFiS3 er inkluderet med Arduino Opla board package

07 C# med MQTTnet

I C# bruger vi MQTTnet-NuGet-pakken til at subscribe og modtage data fra Arduino-enheder. 

// dotnet add package MQTTnet

using MQTTnet;
using MQTTnet.Client;

var factory = new MqttFactory();
var client = factory.CreateMqttClient();

var options = new MqttClientOptionsBuilder()
    .WithTcpServer("localhost", 1883)
    .WithClientId("csharp-dashboard")
    .Build();

client.ApplicationMessageReceivedAsync += e => {
    var topic   = e.ApplicationMessage.Topic;
    var payload = e.ApplicationMessage.ConvertPayloadToString();
    Console.WriteLine($"{topic}: {payload}");
    return Task.CompletedTask;
};

await client.ConnectAsync(options);
await client.SubscribeAsync("skole/+/temp");

Console.WriteLine("Lytter på skole/+/temp — tryk Enter for at stoppe");
Console.ReadLine();
await client.DisconnectAsync();
